안녕하세요 😊 오늘은 오래 준비해 온 배포 소식을 드리러 왔습니다.

예전에 한 번 소개 드렸던 적이 있는데요 — SmartThings 미지원 삼성 시스템 에어컨을 와이파이 킷 없이도 제어할 수 있는 방법이었죠. 에어컨의 RS485 단자에 EW11 WiFi-RS485 브릿지를 연결해서 TCP로 통신하는 방식이었는데, 그걸 이번에 제대로 정리해서 배포 가능한 형태로 개선했습니다.

크게 두 가지 구성 요소가 있어요. 첫 번째는 EW11과 SmartThings Edge Driver 사이에서 중계 역할을 하는 브릿지 서버이고, 두 번째가 SmartThings 앱에서 에어컨을 직접 제어할 수 있게 해주는 Edge Driver입니다. 이 글에서 둘 다 설치하는 방법을 차근차근 안내해 드릴게요.

Part 1

브릿지 서버 설치

브릿지 서버의 소스 코드는 GitHub에 공개되어 있습니다. 자세한 내용은 레포지토리의 README.md를 꼭 함께 읽어보세요.

🔗 GitHub 레포지토리
[레포지토리 URL 추후 추가 예정]

아래는 핵심 설치 흐름을 요약한 내용입니다.

① 파일 배치

서버에 접속한 뒤, 레포지토리를 클론하거나 프로젝트 디렉토리를 복사해 배치합니다. 

# 레포지토리 클론 (또는 scp로 직접 복사)
git clone [레포지토리 URL] /opt/ac-bridge-server
cd /opt/ac-bridge-server

② config.json 설정

설치 디렉토리 안의 config.json을 열어 EW11 IP 주소와 포트를 입력합니다. 서버 포트는 나중에 Edge Driver에서도 똑같이 입력해야 하니 잘 기억해 두세요. 

{
  "server": {
    "host": "0.0.0.0",   // 외부 접속 허용
    "port": 8888        // Edge Driver와 맞춰야 함
  },
  "ew11": {
    "host": "192.168.0.38", // EW11 IP 주소
    "port": 8899          // EW11 기본 포트
  },
  "controller_mode": "real",
  "log_level": "INFO"
}
항목 설명
server.port 브릿지 서버 포트. Edge Driver 설정 값과 반드시 일치해야 합니다.
ew11.host EW11 장치의 IP 주소 (공유기 DHCP 고정 설정 권장)
ew11.port EW11 TCP 포트 (별도 변경하지 않았다면 기본값 8899)
controller_mode real: 실제 EW11 사용 / mock: 테스트용 더미 모드

③ Docker로 실행

# 빌드 후 백그라운드 실행
docker compose up -d --build

# 로그 확인
docker logs -f ac-bridge-server

정상적으로 실행되면 아래와 같은 로그가 출력됩니다.

INFO  AC Bridge Server starting — mode=real
INFO  EW11 connected: 192.168.0.38:8899
INFO  auto-registered unit: label=에어컨 1
INFO  auto-registered unit: label=에어컨 2
💡 실내기 자동 검색
브릿지 서버는 RS485 버스를 모니터링하다가 실내기의 상태 패킷을 수신하는 순간 자동으로 등록합니다. 별도로 주소를 입력할 필요 없고, 서버 시작 후 약 5분 이내에 연결된 모든 실내기가 등록됩니다.
📌 업데이트 방법
설정(config.json)만 바꿨다면 docker restart ac-bridge-server로 재시작만 하면 됩니다. 코드 변경 시에는 docker compose up -d --build --force-recreate로 재빌드하세요.

여기까지 완료하면 브릿지 서버 IP와 포트 번호(기본 8888)를 갖게 됩니다. 이 두 값이 다음 단계 Edge Driver 설정의 핵심입니다.

Part 2

SmartThings Edge Driver 설치 및 설정

브릿지 서버가 준비되었다면, 이제 SmartThings 측 설정 차례입니다. Edge Driver를 허브에 설치하고, 브릿지 서버 정보를 입력해 에어컨을 추가하는 과정을 순서대로 안내합니다.

STEP 2-1 Edge Driver 허브에 설치하기

아래 초대 URL에 접속해 채널에 등록(Enroll)한 뒤, SmartThings 앱에서 Samsung AC Driver를 내 허브에 설치합니다.

🔗 Edge Driver 초대 URL
[초대 URL 추후 추가 예정]

초대 URL 접속 → Enroll Hub → 허브 선택 → Samsung AC Driver 항목에서 Install

STEP 2-2 AC Bridge 장치 추가 및 서버 설정 입력

Driver 설치 후 SmartThings 앱에서 장치 추가 → 주변 검색을 실행하면 Samsung AC Bridge라는 장치가 검색되어 추가됩니다.

📸
장치 추가 → 주변 검색 화면 스크린샷
스크린샷 추후 첨부 예정

AC Bridge 장치가 추가되었으면, 장치 설정(⚙️)으로 들어가서 Part 1에서 확인한 브릿지 서버 IP와 포트를 입력해 주세요.

📸
AC Bridge 장치 설정 화면 — IP / 포트 입력 스크린샷
스크린샷 추후 첨부 예정
✅ 입력 예시
서버 IP: 192.168.0.100  |  포트: 8888 (config.json의 server.port와 동일하게)
STEP 2-3 실내기 장치 추가 및 매핑

AC Bridge 설정을 저장했으면, 다시 장치 추가 → 주변 검색을 실행하세요. 실외기에 연결된 모든 실내기가 자동으로 추가됩니다. 🎉

📸
실내기 장치 자동 검색 결과 스크린샷
스크린샷 추후 첨부 예정
⚠️ 실내기가 일부만 추가되었다면?
브릿지 서버가 아직 모든 실내기의 상태 패킷을 수신하지 못한 상태입니다. 서버 시작 후 5분 정도 기다렸다가 다시 장치 추가를 시도해 보세요.

실내기는 각 실내기의 내부 주소를 기준으로 등록되기 때문에, 처음에는 에어컨 1, 에어컨 2 … 처럼 번호로만 표시됩니다. 실내기를 하나씩 켜보면서 SmartThings 앱에서 어떤 장치가 반응하는지 확인한 뒤, 장치 이름과 방(위치)을 직접 매핑해 주시면 됩니다. 약간 번거롭지만 한 번만 하면 끝이에요 😄
